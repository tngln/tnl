# 开发者工具框架

## 现状更新
- 本文是 Developer 工具窗口的最初框架计划。
- **当前状态：✅ 已实现**
- Developer 窗口已切到 `SurfaceWindow + TabPanelSurface`
- Timeline tab 已移除
- `Control`、`Data`、`Storage` 等面板已经不再只是占位
- 多数高层面板已经迁到 `JSX + defineSurface + Panel*` 范式
- 当前状态与推荐写法请优先参考：`UI系统现状与调用约定.md`

## 目标（已完成）
- 在 `src/ui/windows/developer/` 下建立新的开发者工具框架，用一个更完整的"开发者工具窗口"取代现有 DeveloperWindow。
- 以"面板（Panel）"为单位组织能力，并用 Tab 切换面板。
- 本阶段只搭建框架与骨架实现：把各面板的入口、占位 UI、以及后续 TODO 写清楚；不要求一次性实现全部面板的真实功能。

## 约束与原则
- 代码迁移/重组优先采用复制与移动，避免"在上下文中重新生成整段旧代码"的方式。
- 保持现有交互/快捷键（例如 F2 打开 Developer 工具）的习惯不变，内部替换为新窗口。
- 新框架尽量复用现有基础设施：`TabPanelSurface`、`ViewportElement`、`Surface`、`ControlsSurface`、`TextSurface`、`DividerSurface`。
- 每个面板先提供"可运行的占位 Surface"，并以文本清晰列出该面板后续需要的接口/数据源/实现 TODO。

## 设计概览
### 1) 目录结构（已实现）
- `src/ui/windows/developer/`
  - `developer_tools_window.ts`：新的开发者工具窗口（替代旧 DeveloperWindow）
  - `panels/`
    - `data_panel.tsx`
    - `storage_panel.tsx`
    - `control_panel.ts`
    - `wm_panel.tsx`
    - `worker_panel.tsx`
    - `codec_panel.tsx`
    - `surface_panel.tsx`
    - `inspector_panel.tsx`
  - `index.ts`：统一 export（窗口与面板注册）

### 2) 面板模型（PanelSpec）
- 定义一个轻量结构（放在 `developer/index.ts` 或 `developer/panels/*` 的公共文件中）：
  - `id`：稳定字符串 ID（用于状态保存、日志、后续深链）
  - `title`：Tab 标题
  - `build`：一个函数，接收 `DeveloperContext` 并返回 `Surface`
- 新的 DeveloperToolsWindow 内部使用 `TabPanelSurface` 渲染 Tab + 内容区，并将当前选中面板的 `surface` 作为 Viewport target。

### 3) 依赖注入（DeveloperContext）
- 为了避免面板直接耦合 `main.ts` 的全局变量，定义一个 `DeveloperContext`：
  - `reactivity?: { list?: () => unknown }`（Data 面板将来使用）
  - `storage?: { opfs?: FileSystemDirectoryHandle }`（Storage 面板将来使用）
  - `wm?: WindowControlApi`
  - `workers?: { info?: () => unknown; list?: () => WorkerRuntimeEntry[] }`
  - `codecs?: { info?: () => unknown; list?: () => CodecRuntimeEntry[] }`
  - `surface?: { listLayers?: () => DebugLayerInfo[]; listBlits?: () => DebugBlitInfo[]; setOverlay?: (rect: Rect | null) => void; setPaintFlash?: (on: boolean) => void; getPaintFlash?: () => boolean }`
  - `inspector?: { tree?: () => DebugTreeNodeSnapshot; eval?: (code: string) => unknown }`
  - `docking?: DockingControlApi`

## 替换旧 DeveloperWindow
1. 新增 `DeveloperToolsWindow`（位于 `ui/windows/developer/`），默认仍由 F2 开关。
2. 在 `main.ts` 将 import 从旧 `ui/window/developer_window.ts` 切换到新 `ui/windows/developer/developer_tools_window.ts`（或通过 `developer/index.ts`）。
3. 确认快捷键、打开/关闭、拖动、缩放无回归后，删除旧 DeveloperWindow 文件或改为薄包装（按你的偏好选择其一）。

## 逐步实施步骤（执行顺序）
1. 创建 `ui/windows/developer/` 目录与面板文件，所有面板返回可渲染的占位 `Surface`。
2. 创建 `DeveloperToolsWindow`：内部使用 `TabPanelSurface` 组合面板；Control 面板挂 `ControlsSurface`，其它面板挂 `TextSurface` 占位并写 TODO。
3. 修改 `main.ts`：将 F2 绑定切到新窗口类与 ID；确保窗口注册与渲染链一致。
4. 清理旧代码：删除或兼容旧 DeveloperWindow；保证无重复 ID 与无死引用。
5. 运行校验：类型检查、单测、预览验证（打开 Developer、切 Tab、窗口交互）。
