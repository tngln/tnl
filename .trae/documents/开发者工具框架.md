# 开发者工具（ui/window/developer）框架计划

## 目标
- 在 `src/ui/window/developer/` 下建立新的开发者工具框架，用一个更完整的“开发者工具窗口”取代现有 DeveloperWindow。
- 以“面板（Panel）”为单位组织能力，并用 Tab 切换面板。
- 本阶段只搭建框架与骨架实现：把各面板的入口、占位 UI、以及后续 TODO 写清楚；不要求一次性实现全部面板的真实功能。

## 约束与原则
- 代码迁移/重组优先采用复制与移动，避免“在上下文中重新生成整段旧代码”的方式。
- 保持现有交互/快捷键（例如 F2 打开 Developer 工具）的习惯不变，内部替换为新窗口。
- 新框架尽量复用现有基础设施：`TabPanelSurface`、`ViewportElement`、`Surface`、`ControlsSurface`、`TextSurface`、`DividerSurface`。
- 每个面板先提供“可运行的占位 Surface”，并以文本清晰列出该面板后续需要的接口/数据源/实现 TODO。

## 设计概览
### 1) 目录结构（新增）
- `src/ui/window/developer/`
  - `developer_tools_window.ts`：新的开发者工具窗口（替代旧 DeveloperWindow）
  - `panels/`
    - `data_panel.ts`
    - `storage_panel.ts`
    - `control_panel.ts`
    - `wm_panel.ts`
    - `timeline_panel.ts`
    - `worker_panel.ts`
    - `codec_panel.ts`
    - `surface_panel.ts`
    - `inspector_panel.ts`
  - `index.ts`：统一 export（窗口与面板注册）

### 2) 面板模型（PanelSpec）
- 定义一个轻量结构（放在 `developer/index.ts` 或 `developer/panels/*` 的公共文件中）：
  - `id`：稳定字符串 ID（用于状态保存、日志、后续深链）
  - `title`：Tab 标题
  - `surface`：一个 `Surface` 实例（先占位，后续可升级为复杂 Surface）
- 新的 DeveloperToolsWindow 内部使用 `TabPanelSurface` 渲染 Tab + 内容区，并将当前选中面板的 `surface` 作为 Viewport target。

### 3) 依赖注入（DeveloperContext，先占位）
- 为了避免面板直接耦合 `main.ts` 的全局变量，定义一个 `DeveloperContext`（先做最小可用版本）：
  - `reactivity?: { list?: () => unknown }`（Data 面板将来使用）
  - `storage?: { opfs?: FileSystemDirectoryHandle }`（Storage 面板将来使用）
  - `wm?: { listWindows?: () => { id: string; open: boolean; minimized: boolean }[]; toggle?: (id: string) => void }`
  - `timeline?: { emit?: (event: unknown) => void; events?: () => unknown[] }`
  - `workers?: { list?: () => unknown[] }`
  - `codecs?: { info?: () => unknown }`
  - `surface?: { listLayers?: () => unknown[] }`
  - `inspector?: { eval?: (code: string) => unknown }`
- 本阶段：窗口创建一个空 context（或仅注入能稳定获取的最小字段），各面板仅把“需要什么”写成 TODO，并显示占位信息。

## 面板骨架（本阶段交付内容）
- Data：占位树形 UI（先用文本/简单列表），列出后续需要的“Signal 注册表/作用域”机制 TODO。
- Storage：占位信息，列出后续 OPFS 空间统计/清理的 API 与流程 TODO。
- Control：直接复用 `ControlsSurface`（这是唯一一个本阶段就有实际内容的面板）。
- WM：占位信息，列出后续 WindowManager 接口（列窗、聚焦、关闭、最小化、定位）TODO。
- Timeline：占位信息，列出后续埋点 API、事件缓冲、可视化时间轴 TODO。
- Worker：占位信息，列出后续 Worker registry / 进度上报协议 TODO。
- Codec：占位信息，列出后续 WebCodecs capability/info 收集 TODO。
- Surface：占位信息，列出后续从 Compositor/Surface 收集 draw ops/layer tree 的调试接口 TODO。
- Inspector：占位信息，列出后续 element picking、表达式执行沙箱、安全限制 TODO。

## 替换旧 DeveloperWindow
1. 新增 `DeveloperToolsWindow`（位于 `ui/window/developer/`），默认仍由 F2 开关。
2. 在 `main.ts` 将 import 从旧 `ui/window/developer_window.ts` 切换到新 `ui/window/developer/developer_tools_window.ts`（或通过 `developer/index.ts`）。
3. 确认快捷键、打开/关闭、拖动、缩放无回归后，删除旧 DeveloperWindow 文件或改为薄包装（按你的偏好选择其一）。

## 逐步实施步骤（执行顺序）
1. 创建 `ui/window/developer/` 目录与面板文件，所有面板先返回可渲染的占位 `Surface`。
2. 创建 `DeveloperToolsWindow`：内部使用 `TabPanelSurface` 组合面板；Control 面板挂 `ControlsSurface`，其它面板挂 `TextSurface` 占位并写 TODO。
3. 修改 `main.ts`：将 F2 绑定切到新窗口类与 ID；确保窗口注册与渲染链一致。
4. 清理旧代码：删除或兼容旧 DeveloperWindow；保证无重复 ID 与无死引用。
5. 运行校验：类型检查、单测、预览验证（打开 Developer、切 Tab、窗口交互）。

## 验收标准
- F2 打开的是新的 DeveloperToolsWindow（Tab 结构可用）。
- 9 个面板均可切换且不报错；Control 面板可实际交互。
- 代码组织符合：基础设施留在 `ui/base`，窗口代码留在 `ui/window`，新的开发者工具按 `ui/window/developer` 结构组织。
- 类型检查与现有测试全部通过；预览中窗口交互（拖动/缩放/关闭）无回归。
