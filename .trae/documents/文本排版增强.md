## 目标
- 在 src/core/draw.text.ts 增加“富文本排版 + 渲染”能力：
  - 在给定最大宽度时自动换行（word 尺度优先）。
  - 支持多段不同颜色（以及可切换 Bold/Italic/Underline）。
  - 使用 Intl.Segmenter 进行 word 级分拆，并对必要测量结果做可控大小的 LRU 缓存。
- 保持与现有 draw.ts 的职责边界：draw.text.ts 负责文本的“测量/排版/渲染”，draw.ts 仍负责几何图元与 Text() 单行绘制。

## 范围与非目标
- **范围**
  - 单段或多段富文本（span 级样式），输出按行组织的布局结果（每个 span 被拆成若干 run）。
  - 仅横排（left-to-right）优先；先不做复杂 bidi/竖排。
  - 换行策略：word 优先；单个 word 超过 maxWidth 时退化为 grapheme/字符级拆分。
  - 下划线：以 lineBox/baseline 推导出一条 hairline/指定 thickness 的 Line 绘制。
- **非目标（本次不做）**
  - 段落级对齐（justify 全对齐）、首行缩进、行间距策略集合、选择/光标编辑、富文本解析（Markdown/HTML）。
  - 复杂 OpenType 特性/字距微调（依赖 Canvas measureText 能力上限）。

## 核心类型设计（建议）
- `TextEmphasis = { bold?: boolean; italic?: boolean; underline?: boolean }`
- `RichTextSpan = { text: string; color: string; emphasis?: TextEmphasis }`
- `RichTextStyle = { fontFamily: string; fontSize: number; fontWeight?: number; lineHeight: number }`
  - span 的 bold/italic 通过“基于 style 拼 font string”实现（例如 `italic 600 14px ...`）。
- `RichTextLayoutOptions = { maxWidth: number; wrap?: "word"; align?: "start" | "center" | "end" }`
- `RichTextRun = { text: string; x: number; w: number; spanIndex: number; font: string; color: string; underline?: boolean }`
- `RichTextLine = { y: number; w: number; runs: RichTextRun[] }`
- `RichTextLayout = { lines: RichTextLine[]; w: number; h: number }`

## API 规划（draw.text.ts）
- **基础（已有）**
  - `measureTextWidth(ctx, text, font): number`
  - `measureTextLine(ctx, text, font, lineHeight): { w: number; h: number }`
- **新增：字体拼接**
  - `fontString(base: RichTextStyle, emphasis?: TextEmphasis): string`
    - 统一生成 Canvas font string（weight/italic/size/family），保证整个项目一致。
- **新增：分词**
  - `segmentWords(text: string): string[]`
    - 优先用 `new Intl.Segmenter(undefined, { granularity: "word" })`。
    - 过滤掉 purely whitespace 的 segment（空白作为单独 token 保留或归一化：见“换行规则”）。
    - Intl.Segmenter 不可用时 fallback：按空白分割并保留空白。
- **新增：排版**
  - `layoutRichText(ctx, spans: RichTextSpan[], base: RichTextStyle, opts: RichTextLayoutOptions): RichTextLayout`
    - 产生 line/run；run 之间 x 累积，超过 maxWidth 自动换行。
    - 空白处理：行首不放置空白；连续空白可折叠为单个空格（可配置，默认折叠）。
    - 超长 token：当 tokenWidth > maxWidth 时，退化到 grapheme（`Intl.Segmenter(..., granularity:"grapheme")` 或简单逐字符）拆分。
- **新增：绘制**
  - `drawRichText(ctx, origin: {x:number;y:number}, layout: RichTextLayout, base: RichTextStyle): void`
    - 使用 `ctx.fillText` 绘制 runs（或复用 core/draw.ts 的 Text() 也可，但不强制）。
    - underline：对带 underline 的 run，在 baseline 下方绘制 Line；颜色跟随 run.color。

## 缓存（LRU）设计
- **缓存目标**
  - `measureTextWidth` 的结果：key = `${font}\n${text}` → width:number。
  - 可能的二级缓存：word segmentation 结果：key = text → tokens:string[]（可选，先做 measure 缓存即可）。
- **LRU 需求**
  - 容量可配置（例如 2k～10k 条），超过则淘汰最旧项。
  - 纯 TS 实现：Map + 双向链表节点（或 Map 迭代重插入也可，但要控制性能/行为）。
- **集成方式**
  - draw.text.ts 内部维护默认 LRU 实例，并导出 `configureTextCache({ maxEntries })`（可选）。
  - 允许调用方提供外部 cache（可选，先不暴露）。

## 换行与布局规则（精简可控）
- 主流程：
  - span → segments（word tokens）→ token widths（缓存）→ 组装到 line。
  - 每行宽度 = runs.w 之和；行高 = base.lineHeight（固定）。
  - align（start/center/end）通过在每行绘制时增加 `xOffset` 实现（layout 阶段也可写死 x=0，draw 阶段偏移）。
- 空白规则（默认）：
  - 将任何空白 token 归一化为 `" "`。
  - 行首空白丢弃；行尾空白丢弃。
- 超长 token：
  - tokenWidth > maxWidth：grapheme 拆分，尽量填满当前行，再换行继续。

## 验证方案
- 单元测试（优先）：
  - 分词：中英文/空白/标点的 tokens 合理性（在 Intl.Segmenter 可用环境下）。
  - 换行：给定 maxWidth 产生多行，行宽不超过 maxWidth（允许浮点误差）。
  - 样式：bold/italic 生成 font string 正确；underline run 产出标记。
  - LRU：容量溢出淘汰顺序正确。
- 手工验证：
  - 在 AboutWindow 或新 demo window 中渲染一段包含不同颜色/样式且会换行的文本，观察一致性。
